Eventify API user stories
Current repository scope only

Guest
- As a guest, I can register a new account: POST /api/v1/auth/register
- As a guest, I can log in and receive a JWT token: POST /api/v1/auth/login
- As a guest, I can request a password reset email: POST /api/v1/auth/password/reset
- As a guest, I can confirm a password reset by sending a new password with a reset token: POST /api/v1/auth/password/reset?token=...
- As a guest, I can browse the list of brands: GET /api/v1/brands
- As a guest, I can view a single brand and its events: GET /api/v1/brands/:id
- As a guest, I can browse the event list: GET /api/v1/events
- As a guest, I can filter the event list by supported query parameters: GET /api/v1/events?params

Authenticated user
- As an authenticated user, I can list users: GET /api/v1/users
- As an authenticated user, I can view a user profile: GET /api/v1/users/:id
- As an authenticated user, I can update my own profile: PATCH /api/v1/users/:id
- As an authenticated user, I can delete my own profile: DELETE /api/v1/users/:id
- As an authenticated user, I can view a single event: GET /api/v1/events/:id
- As an authenticated user, I can list categories: GET /api/v1/categories
- As an authenticated user, I can create a category: POST /api/v1/categories
- As an authenticated user, I can view categories linked to an event: GET /api/v1/events/:event_id/categories
- As an authenticated user, I can create a brand and become its owner: POST /api/v1/brands
- As an authenticated user, I can view my tickets: GET /api/v1/tickets or GET /api/v1/my_tickets
- As an authenticated user, I can create a ticket for an event: POST /api/v1/tickets
- As an authenticated user, I can view one of my tickets: GET /api/v1/tickets/:id
- As an authenticated user, I can update one of my tickets, for example to change its active state: PATCH /api/v1/tickets/:id
- As an authenticated user, I can create or update a review for one of my tickets: POST /api/v1/tickets/:id/review

Brand manager
- As a brand manager, I can view the membership list for my brand: GET /api/v1/brands/:brand_id/memberships
- As a brand manager, I can add non-owner members to my brand: POST /api/v1/brands/:brand_id/memberships
- As a brand manager, I can update memberships when policy rules allow it: PATCH /api/v1/brands/:brand_id/memberships/:id
- As a brand manager, I can remove members with the restrictions defined by policy: DELETE /api/v1/brands/:brand_id/memberships/:id
- As a brand manager, I can attach categories to events in my brand: POST /api/v1/events/:event_id/categories
- As a brand manager, I can remove categories from events in my brand: DELETE /api/v1/events/:event_id/categories/:category_id
- As a brand manager, I can submit an event for review: POST /api/v1/events/:id/submit
- As a brand manager, I can cancel an event in my brand: POST /api/v1/events/:id/cancel

Brand owner
- As a brand owner, I can update my brand: PATCH /api/v1/brands/:id
- As a brand owner, I can delete my brand: DELETE /api/v1/brands/:id
- As a brand owner, I can manage all brand memberships, while preserving at least one owner: POST/PATCH/DELETE /api/v1/brands/:brand_id/memberships...
- As a brand owner, I can submit an event for review: POST /api/v1/events/:id/submit
- As a brand owner, I can cancel an event in my brand: POST /api/v1/events/:id/cancel

Superadmin
- As a superadmin, I can create events directly: POST /api/v1/events
- As a superadmin, I can approve events under review: POST /api/v1/events/:id/approve
- As a superadmin, I can reject events under review: POST /api/v1/events/:id/reject
- As a superadmin, I can update or delete any user: PATCH/DELETE /api/v1/users/:id
- As a superadmin, I can update or delete any brand: PATCH/DELETE /api/v1/brands/:id
- As a superadmin, I can manage any brand membership: GET/POST/PATCH/DELETE /api/v1/brands/:brand_id/memberships...

Not currently implemented
- Email confirmation
- Refresh tokens and logout
- Event subscriptions
- Payments
- Brand approval requests
- Dedicated admin moderation endpoints
